# 工业缺陷检测系统 · 整合评测报告

> 数据集:MVTec AD `metal_nut`(金属螺母,真实机加工件)。训练 220 张正常样本,测试 115 张(good + bent/color/flip/scratch)。
> 硬件:CPU only(Windows / Python 3.13)。

## 1. 任务与指标体系

- **任务**:无监督异常检测——仅用正常样本学习正常分布,检测并像素级定位偏离区域。
- **指标**:
  - **Image AUROC**:整图判正常/异常的区分能力。
  - **Pixel AUROC**:像素级缺陷定位精度。
  - **F1**:综合精确率/召回率。
  - **部署指标**:单张推理延迟、吞吐(FPS)、模型体积。
- **工业取向**:漏检代价 >> 误检,实际阈值偏向高召回。

## 2. 模型选型对比(W1–W2)

同品类、同 CPU 下:

| 模型 | Image AUROC | Pixel AUROC | Image F1 | 拟合耗时 | 推理延迟/张 | 模型体积 |
|---|---|---|---|---|---|---|
| **PaDiM** | 0.937 | 0.946 | 0.930 | **35 s** | **169 ms** | 168 MB |
| **PatchCore** | **0.997** | **0.987** | **0.984** | 6200 s (~103 min) | 1422 ms | 227 MB |

**结论**:PatchCore 精度近满分但 CPU 拟合需 ~100 分钟、推理 1.4 s/张,**不适合实时/边缘**;PaDiM 轻量(35 s / 169 ms),精度低一档但工程可落地。**精度优先选 PatchCore,实时/边缘选 PaDiM**。

## 3. 逐缺陷类型分析(W3)

PaDiM,图像级阈值 = good 的 mean+2·std = 0.589:

| 缺陷类型 | 检出率 | 空间性质 | 机械解读 |
|---|---|---|---|
| flip(翻面) | 100% | 全局/整件 | 朝向错误,整件偏离 → 最易检 |
| color(色差) | 64% | 局部 | 局部变色,看对比度 |
| bent(弯曲) | 60% | 局部几何 | 局部塑性变形,看幅度/角度 |
| scratch(划痕) | 61% | 细小局部 | 低对比度细线痕,被大面积正常区稀释 → 最难检 |
| good(正常) | 5%(误报) | — | 误报率低 |

**结论**:高 Image AUROC 掩盖了细小局部缺陷(scratch/bent)在固定阈值下约 40% 漏检——**评估必须分类型看**;小目标缺陷应看 Pixel AUROC、用更强定位模型、或下调阈值换高召回。

## 4. 边缘部署(W4)

PaDiM 导出后,端到端逐张推理(读图→预处理→推理→判定,batch=1):

| 后端 | 平均延迟 | 吞吐 | Image AUROC |
|---|---|---|---|
| PyTorch (CPU) | 189.4 ms | 5.3 FPS | 0.9482 |
| **ONNX Runtime (CPU)** | **132.9 ms** | **7.5 FPS** | 0.9482 |

**结论**:ONNX Runtime 较 PyTorch **≈1.43× 加速,AUROC 逐位一致(零精度损失)**。验证导出正确性时发现"双重预处理"会使分数饱和、AUROC 退化到 0.5,靠**与参考实现逐位对齐**定位。

## 5. VLM 诊断层(W5)

检测→定位→整图+缺陷框+知识库→Qwen2.5-VL(qwen-vl-max)输出类型/成因/建议。两例(bent、color)类型判断均正确,成因合理。

**关键发现**:紧裁剪会丢失整体轮廓,使几何缺陷(bent)被误判为 color;改喂整图+框后正确。**VLM 诊断质量高度依赖输入构造**。幻觉控制:检测先行 + 知识库接地(RAG)+ 结构化输出 + 抽样人工核对。

## 6. 总体结论

| 能力维度 | 体现 |
|---|---|
| 工程化/边缘部署 | ONNX 导出 + CPU 推理基准,1.43× 加速、精度无损,踩坑可复盘 |
| 深度学习/CV | 无监督异常检测,多模型选型,像素级定位,分类型误差分析 |
| 多模态 | 检测+VLM 诊断管线,RAG 接地,幻觉控制 |
| 机械制造场景 | 聚焦金属件,从加工工艺角度解读缺陷与成因(护城河) |

## 7. 局限与下一步

- **数据**:用公开 MVTec,真实产线需域适应微调(域偏移)。
- **小目标缺陷**:scratch 检出偏低,可用更高分辨率输入、patch 级评分、按类型分别定阈值。
- **延迟**:当前为端到端(含预处理),纯前向 FPS 可再单测;Intel 设备上转 OpenVINO IR 可进一步加速。
- **模型**:EfficientAD(需 GPU 训练)待补,作为"轻量实时"与 PaDiM/PatchCore 三方对比。
- **VLM**:接入工艺标准文档做检索增强,给诊断附引用依据;统计类型判断准确率。
